Host your website as github.io

Tutorial on hosting website on github with Hugo

May 8, 2018 CP Tai

2 minute read

  • Twitter

  • Google+

  • Facebook

  • Reddit

  • LinkedIn

  • StumbleUpon

  • Pinterest

  • Email

Introduction

Pre-requisite:

  1. Have hugo installed in your environment
  2. Github account
  3. Basic linux commands

Step 1: use Hugo to create a default folder for the site

>>hugo new site <blog name>

Then you should be able to see a structure like this in / folder

.
├── archetypes
├── config.toml
├── content
├── data
├── layouts
├── static
└── themes

Step 2: Create two repos on github * One with the <blog name>, e.g. my case will be cplog. * And another one is .github.io, with README.md e.g. mycase cptai.github.io.

Step 3: Choose a theme that you like from here, then go to the theme’s github repo and clone it at next step

Step 4: Go to the /theme which is inside /<blog name> [created at step 2] and clone:

>>git clone <the theme github url>

Step 5: go to /</blog name>/theme/<theme name>, you should be able to see the /exampleSite folder, copy the config.toml to replace the one at /<blog name>

Step 6: back to /<blog name> and edit the config.toml, change the info with respect to your blog name

Step 7: At /<blog name> if you type hugo server you should be able to view your website at localhost:1313

Step 8: Before push to github, use hugo command will generate a new folder /<blog name>/public

Git init your website, and push the folder to your <blog name> git repo

Step 9: Remove public from this git because we will put it as submodule and let it belongs to

>>git rm -rf public

Step 10: use hugo command to generate /<blog name>/public again then

>>git submodule add https://github.com/<blog_name>/<blog_name>.github.io.git public

Step 11: git push inside /<blog name>/public again to will publish you website at .github.io

Finally, Open you browser and go to <blog_name>.github.io will see your website!
